#### イミュータブル、ミュータブル、スプレッド構文

JS 使いこなすためにも学ぶのは有意義  
破壊的メソッド？？使わないほうがいい。コードがややこしくなる可能性あるのを回避するため？
元のオブジェクトを破壊するもの。考えが結びつかない・・・
ミュータブル（変更可：配列）、イミュータブル（変更不可：文字列、数値など）
ミュータブルをイミュータブルで扱いたい場合は、スプレッド構文を使う
配列（ミュータブル）をイミュータブルに扱う＝スプレッド構文を使う  
JS だとイミュータブルが推奨
